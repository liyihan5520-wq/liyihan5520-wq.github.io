《Agentic Design Patterns》 (9~16)

2025-12-03 8 min read

第9章： 学习和适应

学习和适应 使 Agent 能够超越预定义参数进行演化，通过经验和环境交互实现自主改进。 通过学习和适应， Agent 可以有效应对新情况并优化其性能，而无需持续的人工干预。

  • 强化学习： Agent 尝试行动并获得积极结果奖励/消极结果惩罚，在动态环境中学习最优行为。适用于控制机器人或游戏角色的 Agent
  • 监督学习：让 Agent 从标注实例中学习输入与期望输出的映射关系，适用于邮件分类或趋势预测的 Agent
  • 无监督学习： Agent 在未标注数据中哦该发现隐藏模式和关联，构建环境心理模型并获取洞察，适用于无特定指导的数据探索场景
  • 基于 LLM 的少样本/零样本学习：利用大语言模型的Agent 通过少量实例，或明确指令快速适应新任务，实现对新命令/情况 的即时响应
  • 在线学习：持续更新知识库以适应动态环境，对实时响应和持续优化至关重要，适用于处理连续数据流的 Agent
  • 基于记忆学习： Agent 会议过往经验调整当前行动，增强上下文感知和决策能力，特别适合具备记忆召回能力的Agent

第10章： 模型上下文协议(MCP)

通过提供标准化接口使 LLM 能与外部资源交互，是实现一致性和可预测性集成的关键机制

function call： 为Agent 提供的一组定制工具，一般内置在模型内部，在上下文中被告知哪些工具可用，模型直接生成调用请求
MCP: 是标准化的通信框架，独立于模型的开放协议层，本身并不提供任何工具，模型通过MCP 客户端请求MCP Server

连接本地 MCP 服务器:

import os
from google.adk.agents import LlmAgent
from google.adk.tools.mcp_tool.mcp_toolset import MCPToolset, StdioServerParameters

## 创建一个可靠的绝对路径，指向名为 'mcp_managed_files' 的文件夹
## 该文件夹位于此 Agent 脚本所在的同一目录中。
## 这确保了 Agent 开箱即用地进行演示。
## 对于生产环境，您需要将此路径指向一个更持久和安全的位置。
TARGET_FOLDER_PATH = os.path.join(os.path.dirname(os.path.abspath(__file__)), "mcp_managed_files")

## 在 Agent 需要之前确保目标目录存在。
os.makedirs(TARGET_FOLDER_PATH, exist_ok=True)

root_agent = LlmAgent(
    model='gemini-2.0-flash',
    name='filesystem_assistant_agent',
    instruction=(
        'Help the user manage their files. You can list files, read files, and write files. '
        f'You are operating in the following directory: {TARGET_FOLDER_PATH}'
    ),
    tools=[
        MCPToolset(
            connection_params=StdioServerParameters(
                command='npx',
                args=[
                    "-y",  # npx 的参数，用于自动确认安装
                    "@modelcontextprotocol/server-filesystem",
                    # 这必须是文件夹的绝对路径。
                    TARGET_FOLDER_PATH,
                ],
            ),
            # 可选：您可以过滤从 MCP 服务器公开的工具。
            # 例如，仅允许读取：
            # tool_filter=['list_directory', 'read_file']
        )
    ],
)

使用 FastMCP 创建 MCP服务器

from fastmcp import FastMCP, Client

## 初始化 FastMCP 服务器。
mcp_server = FastMCP()

## 定义一个简单的工具函数。
## `@mcp_server.tool` 装饰器将此 Python 函数注册为 MCP 工具。
## 文档字符串成为 LLM 的工具描述。
@mcp_server.tool
def greet(name: str) -> str:
    """
    生成个性化的问候语。

    参数：
        name: 要问候的人的名字。

    返回：
        问候语字符串。
    """
    return f"Hello, {name}! Nice to meet you."

## 或者如果您想从脚本运行它：
if __name__ == "__main__":
    mcp_server.run(
        transport="http",
        host="127.0.0.1",
        port=8000
    )

第12章：异常处理和恢复

异常处理和恢复对于构建强大和可靠的 Agent 至关重要，此模式涉及检测错误、优雅地处理错误以及实施恢复策略

第13章: 人机协同

人机协同模式在 Agent 开发和部署中扮演着关键战略角色，它巧妙地将人类认知的独特优势，与AI 的计算能力和效率相结合，通过整合人工智能与人类输入来增强 Agent 能力，其目标并非取代人类输入，而是通过确保关键判断和决策基于人类理解来增强人类能力

  • 人类监督：监控 Agent 性能和输出，确保遵循指南并防止不良结果
  • 干预和纠正: 在 Agent 遇到错误或模糊场景并请求人工干预，人类操作员可纠正错误、提供缺失数据或指导 Agent
  • 学习人类反馈: RLHF 算法直接影响 Agent 的学习轨迹
  • 决策增强： Agent 向人类提供分析和建议，由人类做出最终决定

第14章: 知识检索(RAG)

RAG 使 LLM 能够访问和整合外部信息、实时数据和特定上下文内容，从而显著提高其输出的准确性、相关性和事实基础。

  • 图 RAG(Graph RAG): 是检索增强生成的一种高级形式，它利用知识图谱而不是简单的向量数据库进行信息检索。通过导航结构化知识库中数据实体（节点）之间的明确关系（边）来回答复杂查询。 一个关键优势是它能够综合来自多个文档的碎片化信息答案。通过这些连接，GraphRAG 提供更多上下文准确和细致的响应。

GraphRAG 为复杂问题提供了卓越的上下文推理，但实施和维护成本要高很多，并且可能引入更高的延迟

  • Agentic RAG: 引入一个推理和决策层，以显著增强信息提取的可靠性，其不仅仅是检索和增强，通过引入 Agent ，充当知识的关键守门人和精炼者，主动质疑索引知识的质量、相关性和完整性
  1. Agent 擅长反思和源验证
  2. Agent 擅长协调知识冲突
  3. Agent 可以执行多步推理来综合复杂答案
  4. Agent 可以识别知识差距并使用外部工具

Agentic RAG 虽然功能强大，但 agentic 层引入的复杂性和成本的显著增加

第15章： Agent 间通信(A2A)

Agent2Agent 协议是一个旨在实现不同 AI Agent 框架间通信与协作的开放标准，其确保了互操作性，允许 使用 LangGraph、CrewAI 或 Google ADK 等技术开发的 AI Agent 能够协同工作，无论其来源或框架差异。
A2A 的基础支柱包括：

  • 核心参与者: 用户、A2A 客户端、A2A 服务器
  • Agent 卡片
  • Agent 发现
  • 通信和任务
  • 交互机制及安全性

第16章： 资源感知优化

资源感知优化使智能 Agent 能够在 运行过程中 动态监控和管理计算、时间和财务资源。

  • 动态模型切换： 根据手头任务复杂性和可用计算资源战略性地选择 LLM （轻量/复杂）
  • 上下文修剪和摘要：通过智能摘要和选择性保留交互历史中最相关的信息，最小化提示词 token
  • 主动资源预测： 通过预测额未来工作负载和系统需求来预测资源需求

第18章：安全模式

引导 Agent 的行为和输出，防止有害、有偏见、无关或其他不良响应

  • 模块化和关注点分离：一个单体的、无所不能的 Agent 是脆弱且难以调试的
  • 通过结构化日志记录实现可观测性:
  • 最小权限原则: Agent 应该被授予执行其任务所需的绝对最小权限集
