Asyncio 异步并发

2025-12-02 3 min read

许多 Agent 项目中均使用了 asyn/awiat 等异步并发，在此系统梳理一下 python asynico 原理和使用方法

协程:

又称微线程 Coroutine，一般子程序（函数）都是层级调用，通过栈实现，调用顺序明确。而协程看上去也是子程序，但在执行过程中，在子程序内部可中断，然后转而执行别的子程序，在适当时再返回接着执行.asyncio 是 python 在语言级实现的 **单进程异步协程框架**，使用 **事件循环(event loop)** 驱动 **协程** 的协作式多任务：

一句话概括，让“等IO”的时间片被别的协程拿去干别的活，从而 单线程 也能 高并发

def A():
    print('1')
    print('2')
    print('3')

def B():
    print('x')
    print('y')
    print('z')

如果由协程执行，在执行 A 过程中，可以随时中断，去执行 B， B也可能在执行过程中中断再去执行 A，结果可能是:

        1
        2
        x
        y
        3
        z

但是实际上在A 中是没有调用 B 的，所以协程的调用比 函数调用理解起来要难一些。看起来A、B 的执行有点像多线程，但协程特点在于是一个线程执行的

  • 协程极高的执行效率，由于子程序切换不是线程切换，而是程序自身控制，因此没有线程切换的开销，和多线程相比，线程数量越多，协程优势就越明显
  • 不需要多线程的锁机制，因为只有一个线程，不存在同时写变量冲突，在协程中控制共享资源不加锁，只需判断状态即可，所以执行效率高很多

Asyncio 是Python 3.4 引入的标准库，直接内置了对异步 IO 的支持

async def hello(name):
    # 打印name和当前线程:
    print("Hello %s! (%s)" % (name, threading.current_thread))
    # 异步调用asyncio.sleep(1):
    await asyncio.sleep(1)
    print("Hello %s again! (%s)" % (name, threading.current_thread))
    return name

async def main():
    L = await asyncio.gather(hello("Bob"), hello("Alice"))
    print(L)

asyncio.run(main())

语法糖:

  • async 将一个函数变为 coroutinue 类型，然后可以将 async 函数扔到 asyncio.run() 中执行
  • await 语法可以让我们在 async 函数中方便地调用另一个 async 函数， 这个调用看起来是 串行执行的，但实际上是由 asyncio 内部消息循环控制
  • 在一个 async 函数内部，通过 await asyncio.gather() 可并发执行若干个 async 函数
