GUI-Agent 综述

2025-12-17 3 min read

定义： 模仿人类操纵手机、网页、电脑
独特挑战：

  • 马尔科夫的序列决策问题，收益延迟
  • 网站和应用程序频繁更新，致使在线观测结果不同于过时的离线信息
  • 各种干扰项，如弹出广告、验证、登录请求
  • 页面选项的不确定性，个性化页面、动态选项、搜索结果的随机顺序
  • 不确定的挑战和故障，如网页加载不完整 或者是 对某些网站的临时访问受限

技术路线：
Perception: 感知、解析当前状态
Planning/Decision: 依据用户任务、当前状态和历史行为，推理和制定未来规划，并给出接下来具体动作
1.采用 文本 LLM 执行规划决策任务

由于纯文本大模型拥有更强的逻辑推理能力，可以将当前状态以自然语言描述的形式输入给文本 LLM，让文本 LLM 结合描述，给出实现规划
缺点：使用自然语言描述图片，势必会造成信息损失，难以完全反应真实状态
2.采用闭源 VLM
许多闭源VLM 拥有非常强的视觉感知能力，能够同时将 Perception 和 Planning/Decision 合二为一，直接从当前屏幕状态给出接下来的具体动作
缺点：领域内视觉标识 VLM 可能没见过，难以完全理解/使用 截屏信息
3.微调开源 VLM
使用领域数据集执行 SFT 微调开源 VLM，增强其领域能力
缺点：数据集难以标识、收集

Perception 方法
1.截屏 XML (View Tree)
冗余信息很多，使用前需要进行精简。
国内 App 精度较差，有些界面元素信息不存在，或者位置不准
2.屏幕图片(Screenshot)
OCR、Summarization、Icon Detection& Captioning、Convert the UI Screenshot to XML or Codes via VLM
3.多种信息融合

VLM-driven UI Agents
VLM： 同时完成感知、规划和决策任务

  • 元素定位 (Grounding) 能力
  • 指称 (Referring) 能力
  • 细粒度 OCR 能力： 支持高分辨率
  • UI 界面理解能力, UI相关问答
  • UI 任务执行和推理

Perception + Close VLM:
SoM(Set-of-Mark Prompting)
SoM 利用检测模型，将图像划分为不同粒度界别的区域，并在这些区域上添加一组标记

Open VLM:

  • 针对 GUI 任务特点设计特有的 VLM 结构
    pix2Act(DeepMind)，Auto-GUI(Meta)，CogAgent(Zhipu)，ScreenAI(DeepMind)
  • 使用GUI 任务数据精调通用VLM，不改变其架构
    SeeClick(ShanghaiAI Lab), Ferret-UI(Apple), MobileVLM(XiaoMi)

如何将GUI-Agent 做好?

  • 数据采集的覆盖度和效率是关键挑战
  • 模型泛化性和从错误中恢复能力有待提升
  • 反问、多轮交互、执行效率等
